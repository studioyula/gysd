# 공여사들 창립자 스토리

## 폴더 규칙

- `v1_260605_client`: 고객 공유본. 수정하지 않고 보관한다.
- `v2_260609_working`: 현재 작업본. 새 수정은 여기서 진행한다.
- `archive`: 이전 테스트 파일, 실험 파일 보관용. GitHub/Vercel에는 올리지 않는다.
- `references`: 원본 자료, PSD, 비교 이미지, 참고 SVG 보관용. GitHub/Vercel에는 올리지 않는다.

## 버전 작업 규칙

1. 고객에게 공유한 버전은 폴더명을 `client`로 유지하고 수정하지 않는다.
2. 새 수정이 필요하면 마지막 고객 공유본을 복사해서 새 버전 폴더를 만든다.
3. GitHub/Vercel에는 버전 폴더와 루트 안내 페이지만 올린다.
4. 임시 배포용 복제 폴더는 만들지 않는다.

## Vercel 링크 규칙

- 루트: 버전 선택 페이지
- `/v1_260605_client/`: v1 고객 공유본
- `/v2_260609_working/`: v2 현재 작업본
